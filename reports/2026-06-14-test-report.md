## 2. AI 结构化输出分析

每个 AI 回复中应包含结构化 JSON 事件（`opposed_checks`、`clues_revealed`、`scene_change` 等）。实际结果：

### 2.1 结构化事件返回率

| 类别 | 次数 | 占比 |
|------|------|------|
| JSON 完全为空 `{}` | 9 | **69%** |
| 有事件但无 opposed_checks | 2 | 15% |
| 返回了 opposed_checks 但内容为空 `[]` | 2 | 15% |
| 真正包含非空 opposed_check | 1 | **7%** |

### 2.2 事件类型分布

13 次 AI 生成中，各事件类型出现次数：

| 事件类型 | 出现次数 |
|----------|---------|
| `opposed_checks` | 3 次（2 次为空数组，1 次有效） |
| `npc_state_changes` | 3 次 |
| `clues_revealed` | 3 次 |
| `summary_update` | 3 次 |
| `scene_change` | 3 次 |
| `required_checks` | 1 次 |
| `proposed_state_changes` | 1 次 |

### 2.3 结论

**DeepSeek v4 Pro 对结构化 JSON 输出指令的遵循率约为 31%**（有任意事件），**对抗检定触发率仅 7%**。69% 的 AI 回复完全忽略了结构化输出要求。

## 3. 对抗检定分析

### 3.1 唯一成功触发的一次

```
🛡️ 对抗检定：效力j 的 恐吓(15) vs 拖拉机师傅 的 心理学(50)
NPC 技能 50→HARD 难度
玩家 1d100=83→FAIL | NPC 1d100=27→REGULAR
判定: NPC REGULAR>调查员 FAIL → NPC胜
```

### 3.2 未触发的典型场景

游戏中出现了多次玩家撒谎/忽悠/套话的情节，AI 在叙事中让 NPC 凭"角色扮演逻辑"直接识破，未调用检定：

- "其实我祖上也是我们村的人"（对陈友撒谎）→ AI 未触发检定
- "老一辈就让我找您呀"（对陈友撒谎）→ AI 未触发检定  
- "姓郑啊"（继续伪装）→ AI 未触发检定
- "陈友让我们往北边去的"（对老汉撒谎）→ AI 未触发检定

## 4. 系统表现评估

### 4.1 正常工作的功能

- 房间创建、加入、生命周期管理
- 模组上传与解析（JSON 格式）
- AI 流式生成与实时显示
- 角色卡编辑与保存
- 骰子检定（技能检定、对抗检定）
- AI 诊断日志记录

### 4.2 待改进

1. **对抗检定触发率低**：AI 模型不可靠，需考虑两段式调用或备选方案
2. **前端角色卡**：CoC 7e 车卡流程需继续完善
3. **AI 回复末尾仍有行动建议**：prompt 禁止不彻底

---

## 5. 建议

1. 将对抗检定从"AI 自主判断"改为**两段式调用**：
   - 第一段：纯 JSON 输出，只判断是否需要检定
   - 第二段：根据检定结果生成叙事
2. 或者改用**后端关键词预检 + AI 确认**的混合方案
3. 增加 AI 回复的原始 JSON 存储，便于后续分析
